Что важно понимать

Назначение

Внешняя база проекта до наших изменений.

Где эта ветка в общем процессе

Основная upstream-ветка для сравнения и точечного переноса идей.

Правило: если ветка не помечена как stable или active implementation, её нельзя использовать как основной источник установки.

Технический профиль

Зависит от upstream-состояния; не является утверждённым профилем нашего форка.

Почему это важно

Мы разделяем stable, implementation, historical и upstream-ветки, чтобы не смешивать рабочую установку, эксперименты, архитектурные заметки и внешние источники.

Состояние ветки

Уже сделано

  • Ветка помечена как read-only.
  • Роль upstream отделена от stable и implementation-контуров.

Риски и ограничения

  • Прямое смешивание upstream и fork-кода может сломать исправленные сценарии.
  • Upstream README может не соответствовать нашему fork-flow.

Как проверять

  • Перед переносом идеи: diff, code review, staging, regression.
  • Без тестов upstream-изменения не попадают в main.

Что нельзя делать с этой веткой

  • Не пушить в upstream/*.
  • Не использовать как документацию нашего форка.
  • Не заменять наш README upstream-текстом.
