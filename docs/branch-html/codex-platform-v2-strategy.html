Стратегия platform-v2

Идея доработанного форка: multi-transport платформа с anti-DPI hardening, fallback и управляемым rollout.
ACTIVE STRATEGY К индексу веток Overview Progress

Идея

  • Не один магический протокол, а управляемая local-first платформа.
  • Цель: невидимость до детекта, устойчивость после деградации, быстрый fallback.
  • Каждый этап проходит design, staging, field acceptance, docs, commit and push.

Технологии

  • REALITY/Vision как быстрый low-latency профиль.
  • REALITY/XHTTP как более агрессивный anti-DPI профиль.
  • Controlled tuning: mobile-safe, low-latency, aggressive-stealth.
  • Hardened AWG как будущий opt-in модуль, не дефолтный контейнер.

Threat Model

  • ordinary DPI: TLS/SNI/ALPN/HTTP fingerprint.
  • active probing: публичный 443 должен давать правдоподобный fallback.
  • single-IP co-tenancy: Xray и AWG на одном IP требуют отдельной приёмки.
  • mobile whitelist: LTE может требовать отдельный fallback-класс.

Ближайший фокус

  • Закрыть Block C1: stealth-xray vs stealth-xhttp.
  • Разделять причины сбоев: DPI, co-tenancy, whitelist, IP/ASN.
  • Собрать production-рекомендацию: main profile, DPI fallback, mobile whitelist fallback.

После C1

  • Ввести stealth-multi только из проверенных transport-путей.
  • Усилить admin/subscription контур.
  • Подготовить hardened-awg на резервном контуре.

Отложено

  • White IP / cloud allowlist research для mobile whitelist windows.
  • Yandex Cloud / VK Cloud / специальные подсети проверять позже по актуальным источникам.
  • Performance tuning открыть отдельным этапом после transport/security.
