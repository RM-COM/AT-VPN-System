## Практическая матрица выбора профиля

### 1. `classic / classic-xray`

- [2026-04-09 00:15:49] Использовать как основной стабильный baseline и официальный install-source.
- [2026-04-09 00:15:49] Выбирать, когда приоритет — предсказуемость, уже закрытый regression-маршрут и минимальный риск для production.
- [2026-04-09 00:15:49] Не считать его главным anti-DPI режимом следующего этапа; это baseline для сравнения и fallback.

### 2. `stealth / stealth-xray`

- [2026-04-09 00:15:49] Использовать как первый stealth-режим с минимальной дополнительной сложностью поверх уже подтверждённого stealth ingress.
- [2026-04-09 00:15:49] Рекомендовать как низколатентный stealth-вариант, когда нужен максимально простой transport path при уже правдоподобной входной схеме.
- [2026-04-09 00:15:49] Считать первым кандидатом для сценариев, где важно быстрое подтверждение работы и меньше внутренних moving parts.

### 3. `stealth / stealth-xhttp`

- [2026-04-09 00:15:49] Использовать как второй stealth-режим с другим поведенческим transport-профилем.
- [2026-04-09 00:15:49] Рекомендовать как следующий кандидат для более агрессивной среды, когда нужен не просто тот же stealth ingress, а другой transport path за ним.
- [2026-04-09 00:15:49] Не объявлять его автоматически лучше `stealth-xray` до клиентских и long-running проверок; сейчас он подтверждён как runnable и verify-clean, но не как безусловно лучший режим для всех сетей.
